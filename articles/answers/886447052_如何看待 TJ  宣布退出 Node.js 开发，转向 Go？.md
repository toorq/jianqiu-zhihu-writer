# 如何看待 TJ  宣布退出 Node.js 开发，转向 Go？

> 回答：2019-11-08 11:49｜更新：2019-11-23 21:47｜赞同：0｜评论：0
> 原文：https://www.zhihu.com/question/24373004/answer/886447052

我司两条腿走路。

涉及到web业务的，全部node。

涉及到高性能、底层、分布式计算的，全部go或者传统一点点的C/C++。

后端组和底层组分别是两个技术team。

两个组技术互相交叉。

后端工程师要求掌握：node 和 go。

底层工程师要求掌握：go、node和C/C++。

整个开发哲学上比较喜欢用微服务。

至于为什么？

因为我是老大，我说了算。

我喜欢UNIX的小而美，KISS原则，微服务，特烦大框架，哈哈哈。

用node写代码让我有一种写kernel 驱动的感觉：）

# Visual Studio 是不是坑了一代人？为什么？

> 回答：2025-01-21 20:47｜更新：2025-01-21 22:17｜赞同：0｜评论：0
> 原文：https://www.zhihu.com/question/57853611/answer/84237524145

---

作为一个从2000年走来的Linux程序员，在那个Windows如日中天的年代，

无奈我当时年轻，一身反骨，觉得，用IDE写代码太不酷了，

我就不用ide，我就要用编辑器+gcc编译器+makefile，

我们这些嵌入式程序员都干到什么程度呢，

基本上写代码是不用调试器的，有bug都是对着代码硬啃，

实在不行加几个打印，但凡打印加多了都是你水平不够。

那个时候太年轻，觉得这样写出来的代码更酷。

* * *

到现在人到中年，觉得自己当时真是中二，

能用ide绝对用ide，写程序着怎么快怎么来，

用emacs，vi，写出的来代码，和用vs等ide写出来的代码跑起来是一样的，

真正决定水平的代码本身，甚至不在代码上，

而是后面的数学，设计思想。

古人说，看是写诗，实际上功夫在诗外，大概就是这个道理。

同样的道理，Mac敲出来的代码也并不比Win敲出来的香，不要追求装东西的盒子，而要追求东西本身，这才是道。

* * *

IDE生产率高才是王道。

VS真香，C#也有可取之处。

（不过这种简单的工具确实对提升代码水平，很有帮助，比较容易touch到底层，知道下面的原理是什么。）
